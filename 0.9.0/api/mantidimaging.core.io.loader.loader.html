mantidimaging.core.io.loader.loader module¶

mantidimaging.core.io.loader.loader.load(input_path=None, input_path_flat=None, input_path_dark=None, in_prefix='', in_format='tif', dtype=<class 'numpy.float32'>, cores=None, chunksize=None, parallel_load=False, file_names=None, indices=None, construct_sinograms=False, progress=None)[source]¶

Loads a stack, including sample, white and dark images.

Parameters

  • input_path – Path for the input data folder

  • input_path_flat – Optional: Path for the input Flat images folder

  • input_path_dark – Optional: Path for the input Dark images folder

  • in_prefix – Optional: Prefix for loaded files

  • img_format – Default:’fits’, format for the input images

  • dtype – Default:np.float32, data type for the input images

  • cores – Default:None (max available), cores to be used if parallel_load is True

  • chunksize – Default:None (auto calculated), chunk of work per worker

  • parallel_load – Default: False, if set to true the loading of the data will be done in parallel. This could be faster depending on the IO system. For local runs (with HDD) recommended setting is False

  • file_names – Use provided file names for loading

  • indices – Specify which indices are loaded from the found files. This DOES NOT check for the number in the image filename, but removes all indices from the filenames list that are not selected

  • construct_sinograms – The loaded images will be used to construct the sinograms during loading

  • progress – The progress reporting instance

Returns

a tuple with shape 3: (sample, flat, dark), if no flat and dark were loaded, they will be None

mantidimaging.core.io.loader.loader.read_in_shape(input_path, in_prefix='', in_format='tif', data_dtype=<class 'numpy.float32'>, cores=None, chunksize=None)[source]¶
mantidimaging.core.io.loader.loader.read_in_shape_from_config(config)[source]¶

This function is intended for internal usage.

Read in ONLY the first image in the specified directory, and return the total shape that all the images in that folder will have after loaded. This is determined by the number of images, and the loaded image’s width and height.

It is assumed all images are the same. If they are not the loader will fail on runtime.

Parameters

config – The reconstruction config from which the parameters are read.

Returns

The full shape of the images in the specified directory in a tuple of (Length, X, Y)
