Expected behaviour for the filter is to be done in place without using more memory.

In reality the memory is increased by about 40MB (4 April 2017), but this could change in the future.

The reason why a 10% window is given on the expected size is to account for any library imports that may happen.

This will still capture if the data is doubled, which is the main goal.
